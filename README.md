# Hexo Notion i18n Blog

This project recreates the style of `jiamin-lin/jiamin-lin.github.io` as source code:

- Hexo `8.1.2`
- NexT `8.11.1`
- Muse scheme
- Local search, Atom feed, archives, tags, categories
- Notion to Markdown sync
- `zh-cn`, `en`, and `ja` language routes
- GitHub Pages deployment through GitHub Actions

The reference site was generated by Hexo `6.3.0`; this project keeps the same NexT.Muse look while using Hexo `8.1.2` to avoid known dependency advisories in the old Hexo line.

## Local Setup

```bash
npm ci
cp .env.example .env
npm run sync:notion
npm run build
npm run server
```

Open <http://localhost:4000>.

## Notion Setup

Create a Notion integration and share the source database/page with it.

Add these GitHub repository secrets:

- `NOTION_TOKEN`
- `NOTION_DATABASE_ID` or `NOTION_ROOT_PAGE_ID`

The provided default Notion id is:

```text
be1e22c687994c7bbfeb6b787384015e
```

Expected Notion database fields:

- `Name` or `Title`
- `Status`
- `Slug`
- `Lang`
- `Tags`
- `Categories`
- `Published`
- `Updated`
- `Summary`
- `TranslationGroup`

For i18n, set `Lang` to one of:

- `zh-cn`
- `en`
- `ja`

Set the same `TranslationGroup` on translated versions of the same article.

## GitHub Pages

In GitHub repository settings, set Pages source to **GitHub Actions**.

Push to `main`. The workflow will:

1. Install dependencies.
2. Sync Notion if `NOTION_TOKEN` exists.
3. Build the Hexo site.
4. Deploy `public/` to GitHub Pages.

For this user Pages repo, keep:

```yaml
url: https://jiamin-lin.github.io
root: /
```

For a project Pages repo, change `_config.yml`:

```yaml
url: https://USERNAME.github.io/REPO
root: /REPO/
```
